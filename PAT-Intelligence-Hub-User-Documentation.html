💰 Financial Analytics Dashboard — Guide

ติดตาม KPI ด้านการเงิน และวิเคราะห์รายรับ-รายจ่ายของ 4 สถานีบริการ

📊 ส่วนประกอบหลัก

1️⃣ KPI Cards (บนสุด)

แสดง 5 KPI สำคัญด้านการเงิน

KPI ความหมาย หน่วย เป้าหมาย FY2569
YTD Revenue รวมรายรับจาก 4 สถาน ล้านบาท (M) 7,700
YTD Expense รวมรายจ่ายจาก 4 สถาน M 4,500
Profit Margin กำไร ÷ รายรับ × 100 % 35-42%
Port Comparison เปรียบเทียบรายรับ: ทกท. vs ทลฉ. vs ทรน. M -
Forecast Accuracy ความแม่นยำ Forecast vs Actual % >90%

2️⃣ Revenue Breakdown Chart

รายรับแบ่งตามประเภท (Fix Fee, Storage, Services, etc.)

💡 What to Look For: ประเภทไหนขึ้นมากที่สุด? ประเภทไหนตก? วิเคราะห์เหตุเพราะ

3️⃣ Trend Chart (7 Months YTD)

แสดงแนวโน้มรายรับ-รายจ่าย-กำไร เป็น Line Chart

📈
Cumulative Mode
แสดงสะสมตั้งแต่ต้นเดือน (Month 1-7)
📊
Monthly Mode
แสดงรายเดือน + เดือน 10 ไฮไลต์ (Bonus ประจำปี)
4️⃣
4-Year Overlay
เปรียบเทียบกับปี 2566, 2567, 2568

4️⃣ Port Drill-Down

เลือก Port (ทกท. / ทลฉ. / ทรน. / กทท.) แล้วดูรายละเอียด

Port ชื่อเต็ม Revenue Model
ทกท. ท่าเรือกรุงเทพ Dwell Time → Storage Revenue (ความเป็นสัดส่วน)
ทลฉ. ท่าเรือแหลมฉบัง Fix Fee (ส่วนใหญ่) + Transshipment
ทรน. ท่าเรือระนอง Fix Fee เล็กน้อย + Regional Trade
กทท. สำนักงานใหญ่ รวม / บริหาร

5️⃣ Special Insights — ทกท. "Dwell Time Chain"

สิ่งสำคัญที่สุด: ทกท. มีรูปแบบเงินกำไรที่ขึ้นอยู่กับ Dwell Time (เวลาคงท่า) มากกว่า Throughput

Chain A: FCL Dwell ↑ → Storage Revenue ↑ → Repair Costs ↑ → Profit Pattern

→ ดูแนวโน้ม Dwell Time ใน Dashboard ด้วย ไม่ใช่แค่ Revenue

📊 MOU Performance Dashboard — Guide

ติดตาม 21 KPI ตามสัญญาวัตถุประสงค์ปี 2569 และการคาดการณ์ (Forecast)

📋 โครงสร้าง MOU

MOU ประกอบด้วย 21 KPI แบ่งเป็น 3 หมวด:

หมวด น้ำหนัก จำนวน KPI อธิบาย
หมวด 1 15% 3 KPI ประสิทธิภาพปฏิบัติการ (Operational)
หมวด 2 45% 9 KPI การบริหารจัดการ (Management)
หมวด 3 (SEPA) 40% 9 KPI เป้าหมายอื่นพิเศษ (Strategic)

🎯 ระดับคะแนน (KPI Scoring)

ระดับ 5 ขั้น

🟢
Level 5 (Excellent)
Actual ≥ Target (ถึงเป้าหมายขึ้นไป)
🟢
Level 4 (Good)
Actual 80-100% ของ Target
🟡
Level 3 (Fair)
Actual 60-80% ของ Target
🟡
Level 2 (Poor)
Actual 40-60% ของ Target
🔴
Level 1 (Critical)
Actual < 40% ของ Target

⚙️ วิธีคำนวณ Score

Linear Interpolation (ส่วนของระยะ)

หาก Min=50, Max=100, Actual=75: Score = 1 + (75-50)/(100-50) × 4 Score = 1 + 0.5 × 4 = 3.0
💡 Note: ทุก KPI ต่ำสุด Score = 1.0 (ไม่ต่ำกว่า) • ทศนิยม 4 ตำแหน่ง

📊 ส่วนประกอบ Dashboard

1️⃣ Overall Score Card

Item ความหมาย
Current Score คะแนนปัจจุบัน (Actual ปัจจุบัน)
Forecast Score คะแนนที่คาดการณ์ (แบบ Manual)
Target Score เป้าหมายปี 2569 = 3.75
Status On Track / At Risk / Off Track

2️⃣ KPI Data Entry Table

กรอกข้อมูล Actual ของแต่ละ KPI (ประจำไตรมาส)

✅ Pro Tip: ข้อมูลจาก 4 Port แล้ว Aggregate ขึ้นมา → Overall MOU Score

3️⃣ Forecast Tab (คาดการณ์)

กรอก Forecast คะแนน สำหรับ Month ที่จะมา (Manual Input)

Q1 (Month 1-3): Forecast 3.6 Q2 (Month 4-6): Forecast 3.7 Q3 (Month 7-9): Forecast 3.8 Q4 (Month 10-12): Forecast 3.9

4️⃣ Gap Analysis

แสดง KPI ที่ต่ำกว่า 3.0 (เป้า Minimum) → ต้องมีแผนปรับปรุง

KPI Current Score Target (3.75) Gap Action Plan
2.7 (ตัวอย่าง) 2.54 3.75 -1.21 มี 4 โปรแกรมปรับปรุง

📑 Data Dictionary

อธิบายศัพท์เทคนิค สมการ และฟิลด์ข้อมูลทั่วไปใน PAT Intelligence Hub

📊 Key Metrics

Financial Metrics

Term (ศัพท์) ความหมาย สูตร หน่วย
Revenue (รายรับ) เงินที่เก็บได้จากการให้บริการท่าเรือ Fix Fee + Variable Fee Million Baht (M)
Expense (รายจ่าย) ค่าใช้สอย เงินเดือน ค่าซ่อม Operating Cost + Staff Cost M
Profit (กำไร) ส่วนเกิน ของรายรับเกินรายจ่าย Revenue - Expense M
Profit Margin อัตราส่วนกำไรต่อรายรับ (Profit / Revenue) × 100 %
YTD ปีทางการ (Year To Date) 1 ม.ค. - สิ้นเดือนปัจจุบัน Cumulative from Jan-current month -
Dwell Time เวลาที่คอนเทนเนอร์คงท่าที่ท่า (ทกท.) Check-In to Check-Out Days Days

MOU KPI Metrics

📈 Scoring Formulas

Linear Interpolation (MOU KPI)

สูตร:

ถ้า Actual < Min: Score = 1.0 ถ้า Actual ≥ Max: Score = 5.0 ถ้า Min ≤ Actual < Max: Score = 1 + ((Actual - Min) / (Max - Min)) × 4 แล้ว Round ไป 4 ทศนิยม

Example: Container Volume KPI

Min = 500K TEU Max = 600K TEU Actual = 540K TEU Score = 1 + ((540-500) / (600-500)) × 4 = 1 + (40/100) × 4 = 1 + 1.6 = 2.6000

❓ คำถามที่พบบ่อย (FAQ)

🏠 Homepage & Hub

Q: ฉันจะเข้า Hub ได้อย่างไร?

A: ไปที่ https://pdee-hub.github.io → เข้า Google Sign-In ด้วยอีเมล @port.co.th → ตรวจสอบ PIN (สำหรับ Financial)

Q: ทำไมต้องมี PIN สำหรับ Financial Dashboard?

A: เนื่องจาก Financial Dashboard มีข้อมูลอ่อนไหว (รายรับ-รายจ่าย) → ต้อง Double Authentication ด้วย PIN (55039)

Q: Dark Mode vs Light Mode — ควรใช้อันไหน?

A: ส่วนตัว: ใช้ Dark Mode ตอนค่ำ + Light Mode ตอนกลางวัน → ป้องกันการเหนื่อยตา

💰 Financial Dashboard

Q: ข้อมูล Financial มี lag นานหน่อย ทำไม?

A: เนื่องจาก Hub ดึงข้อมูลจากระบบ Accounting ต้นฉบับที่อัปเดต 1-2 วันทีละครั้ง → ปกติจะมี lag นี้

Q: Dwell Time คืออะไร? ทำไม ทกท. ต้องสนใจ?

A: Dwell Time = เวลาที่คอนเทนเนอร์ค้างท่า → เท่านาน ยิ่งมีค่า Storage Revenue มาก → ทกท. ขึ้นกำไรเมื่อ Dwell Time นาน

Q: ทำไม ทลฉ. มี Fix Fee ส่วนใหญ่ แล้ว ทกท. ไม่?

A: Business Model ต่างกัน: ทลฉ. = สัญญาระยะยาวกับ Shipping Line → Fix Fee • ทกท. = คูลี่-ตามสินค้า → Variable Fee

📊 MOU Performance

Q: Score 1-5 คิดอย่างไร? ทำไมไม่ใช้ 0-100?

A: ตามมาตรฐาน สคร. → ใช้ 5-point Scale → และ Linear Interpolation (ไม่ Jump ทันใจ)

Q: ทำไม Minimum Score = 1.0 ไม่ใช่ 0?

A: ตามหลักการ: แม้ Actual น้อยมาก ยังต้องให้เครดิต Effort → Score ต่ำสุด = 1.0 (ยังมีความพยายาม)

Q: Forecast คืออะไร? ต้องกรอกไหม?

A: Forecast = ค่าที่คาดการณ์ว่าจะถึง (Manual Input) → ใช้สำหรับวางแผน / เตรียมใจ → ไม่บังคับ แต่ดีที่สุดให้กรอก

⚙️ CBE Enablers

Q: Maturity Gating 0-5 แล้ว Sub-Item ก็มีเยอะ — ต้องกรอกทั้งหมดไหม?

A: ใช่ → ทั้ง 10 มิติ × 4-16 Sub-Item ต่อมิติ → ประมาณ 80-150 Sub-Items รวม → แต่ยังค่อย ๆ ทำไป

Q: Best Practice แล้วแต่ SAP? Guideline ไหน?

A: Best Practice = ตามมาตรฐาน SEAM Manual 2569 สคร. + Feedback Reports ที่ปังมาก 3 ปี

Q: Evidence needed = อะไร? ต้องนำเสนออะไร?

A: Evidence = เอกสารพยาน (Policy / Process / Report / Training) → ใช้สำหรับ TRIS Presentation → ยิ่งมี ยิ่งดี

🔗 Linkage Map

Q: Linkage Map ใช้ทำอะไร? ทำไมสำคัญ?

A: แสดง 10 มิติไม่ใช่เดี่ยวๆ → มีการเชื่อมโยง → Knowledge Management ← Digital Transformation ← Knowledge ← Enabler chain
